# 专题重排规则（通用）

适用范围：`topics/*.md`

目标来源：用户给定的目标分类文档（例如 `artifacts/list/output/dp.md`）。  
重排时只学习分类结构，不照搬原文叙述。

## 1. 层级映射
1. 保留专题文件顶层骨架：`# 标题`、`## 概览`、`## 核心思想`、`## 解题流程`、`## 模板与子方法`（如有 `## 易错点` / `## 总结` 可保留）。
2. 在 `## 模板与子方法` 下：
   - `###` 对应目标分类文档中的大分类（原文 `##` 级）。
   - `####` 对应大分类下的二级小分组（原文 `###` 级）。

## 2. 标题整理规则
1. 去掉原文中的序号/节号前缀（如 `一、`、`§1.1`）。
2. 标题使用纯语义文本，不手写任何编号（包括中文序号、`§` 前缀、`4.1 / 4.1.1` 这类数字编号）。
3. `####` 标题可保留必要限定词（如“选做”）仅在确有区分价值时使用。
4. 标题编号（如需）只能由 `scripts/normalize_topics.py` 自动维护，Agent 不得手工参与编号。

## 3. 小分组固定结构
每个 `####` 小分组只保留以下内容（按顺序）：

1. `模板：`
2. 模板代码块（语言标识正确，如 `python`）
3. `模板题目：`
4. 若干题目条目（使用统一链接格式）

不保留以下区块：`思维扩展`、`答疑`、`讲解`、`前言`、`思考题` 等叙述性内容。

## 4. 题目条目格式
严格使用以下格式：

`- 0000 - 题目名 ｜ [LeetCode 链接](https://leetcode.cn/problems/xxx/) ｜ [题解笔记](../solutions/xxxx-xxxx/xxxx-slug.md)`

约束：
1. `id` 必须四位补零。
2. 题目名必须与对应 `solutions/*.md` 的 H1 中文名一致。
3. `LeetCode` 链接必须与 `题解笔记` 文件名中的 slug 对齐。
4. `题解笔记` 路径必须真实存在。
5. 同一题号在全量 `topics/*.md` 中不能重复。

## 5. 排序与裁剪
1. 大分类与小分组顺序按目标分类文档中的出现顺序。
2. 小分组内的题目建议按“模板优先、基础到进阶”组织；若无明确层级，保持现有顺序并逐步整理。
3. 仅收录仓库已存在题解链接；缺失题解不添加占位条目。
4. 若某个小分组当前没有可链接题目，允许暂时保留为空分组（留待后续补充）。

## 6. 章节覆盖完整性
1. 迁移时必须完整保留目标分类文档中的章节架构（大分类与二级小分组都要出现）。
2. 允许“有结构但暂时无题目”，但不允许直接省略该分类节点, 如果分类下没有任何题目, 写为这个格式:
```markdown
#### 4.12.3 树上最小支配集
待补充...

### 4.19 专题：跳跃游戏
待补充...

### 4.20 其他
待补充...
```


## 7. 原有内容迁移规则
1. 迁移不仅包含题目链接，也包含原笔记中已有的说明性内容（如题解思路、方法说明、注意事项等）。
2. 若原内容可以明确匹配到新的 `###` / `####` 标题，必须完整迁移到对应新位置，不得擅自删减。
3. 若原内容无法与新结构建立明确对应关系，可暂不迁移（避免误挂到错误分类）。

## 8. 变更范围约束
1. 仅允许修改用户明确指定的专题文件（例如只指定 `topics/dynamic-programming.md` 时，不得改动其他 `topics/*.md`）。
2. 若格式化脚本会批量改动其他文件，禁止直接执行全量格式化；应改为仅处理目标文件，或先征得用户明确同意。

## 9. 完成后的检查
1. 对目标文件执行去编号与空行规范化（仅处理用户指定文件，避免全量改动）。
2. 运行 `python scripts/ci/check_topics.py`。
3. 如涉及 `solutions` 关联改动，额外运行 `python scripts/ci/check_solutions.py`。
